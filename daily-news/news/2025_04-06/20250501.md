### security report
- [Ransomware debris: an analysis of the RansomHub operation](https://www.group-ib.com/blog/ransomware-debris/)
    - RansomHubは、**Ransomware-as-a-Service (RaaS) グループ**として運営されていました。
    - 2024年2月に出現し、Knight（以前のCyclops）のコードを引き継いだ可能性が指摘されており、競合他社への操作やExit Scamの時期に**経験豊富なアフィリエイト**を多数引き付けました。
    - アフィリエイトに対して、当初は**業界標準より大幅に低い10%という手数料率**を提示していました（後に15%に上昇）。
    - 提供するランサムウェアは、**Windows, Linux, FreeBSD, ESXi**といった**多様なOSとアーキテクチャ**に対応していました。
    - ローカルファイルシステムに加え、**SMBやSFTPを介したリモートのファイルシステム**の暗号化が可能です。
    - アフィリエイト向けに**専用のウェブパネル（ビルダー）**が提供され、ランサムウェアの設定や被害者との交渉を管理できるようになっていました。
    - 攻撃対象には特定の制限があり、CIS諸国や一部の国への攻撃は禁止され、後に**政府機関への攻撃も厳格に禁止**されました。
    - 恐喝手法として、企業収益に基づいた身代金計算や、**GDPRなどの規制当局へのインシデント通報を脅迫**する戦術が指南されていました。
    - 被害者にはランサムノートを通じてライブチャットへのアクセス情報が提供され、アフィリエイトはパネル上で**交渉状況を確認・共有**できました。
    - **2025年4月1日からこのオペレーションは停止**しており、同時期にQilinの活動が活発化したことから、一部の犯罪者が**Qilinに移行した可能性**が示唆されています。
- [Building Own MCP - Augmented LLM for Threat Hunting](https://tierzerosecurity.co.nz/2025/04/29/mcp-llm.html)
    - この投稿は、脅威ハンティングのために MCP (Model Context Protocol) を利用した Augmented LLM の構築について解説している。
    - MCPは、Anthropicが2024年11月に導入した、AIモデルを外部ツールやデータソースに接続するためのオープンスタンダードである。
    - MCPは、ホストアプリケーション(クライアント)が複数のMCPサーバーに接続するクライアント-サーバーアーキテクチャで機能する。
    - MCPサーバーは、ファイルのようなデータを提供する「リソース」、LLMが呼び出し可能な「ツール」、特定のタスクを支援する「プロンプト」の3種類の機能を提供できる。
    - MCPを使ったAIワークフローを開発する際には、新たな攻撃経路を生む可能性があるため、初期段階からセキュリティを考慮することが極めて重要である。
    - 主要なセキュリティ考慮事項には、ユースケースとリスクの理解、セキュアなアーキテクチャと実装、MCPサーバーおよびクライアントのセキュリティ対策が含まれる。
    - 特に、コマンドインジェクションやSQLインジェクションを防ぐための入力検証とデータサニタイズ、そして機密情報の安全な管理が強調されている。
    - プロンプトインジェクションやガードレールバイパスといったAI固有の脆弱性への対策も必要である。
    - 著者は、Sysmonログを収集・格納したELKスタックと脅威インテリジェンスを活用するため、Claude DesktopをクライアントとするローカルMCPサーバー環境を構築した。
    - 構築されたAugmented LLMは、資格情報ダンプの検出、既知の悪意のあるファイルのIOC照合、ログの要約といった脅威分析タスクで有効性を示した。
