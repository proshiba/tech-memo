#### Microsoft：SesameOpマルウェア、OpenAI Assistants APIを悪用した攻撃を確認
https://www.bleepingcomputer.com/news/security/microsoft-sesameop-malware-abuses-openai-assistants-api-in-attacks/

- 要約
    - Microsoftは新たなバックドア「SesameOp」がOpenAI Assistants APIをC2として悪用する攻撃を確認したと発表。
    - 同社DARTは2025年7月の侵害調査で発見し、攻撃者は正規クラウド経由で長期的に遠隔操作と持続化を実現。
    - コンポーネントはAPIから圧縮・暗号化コマンドを取得し復号・実行、収集情報も暗号化して同チャネルで送信。
    - .NET AppDomainManagerインジェクションでVisual Studio関連ユーティリティに展開し、内部Webシェルや常駐プロセスで長期スパイ活動。
    - OpenAIの欠陥ではなく機能の悪用で、MicrosoftとOpenAIは不正アカウントとAPIキーを特定し停止したと説明。

    - エンドポイントでTamper Protectionを有効化し、検出をブロックモードで構成。
    - 上記の継続的監視と設定強化を組み合わせて影響を緩和。

#### Microsoft：WSUS脆弱性の修正パッチがWindows ServerのHotpatchを無効化
https://www.bleepingcomputer.com/news/microsoft/microsoft-patch-for-wsus-flaw-disabled-windows-server-hotpatching/

- 要約
    - WSUSのRCE脆弱性CVE-2025-59287に対処するOOB更新KB5070881が、一部のWindows Server 2025でHotpatchの登録状態を失わせた。
    - CVE-2025-59287について、NCSC-NLは悪用の事実とPoC公開を確認。CISAはKEVに追加し政府機関へ早期対処を指示、Shadowserverは8530/8531公開を約2,600件追跡。
    - 影響はHotpatch登録済みのWindows Server 2025/VMに限定。Microsoftは該当端末へのKB5070881提供を停止し、11～12月のHotpatch受信不可と説明。
    - 未展開ならKB5070893へ切替でHotpatchを維持可能。Windows Updateで一時停止→再開し再スキャン、10月の基盤KB5066835上に提供される。
    - CVE対策の一環として、WSUSのエラー報告に表示される同期エラー詳細は非表示化。影響端末は当面、再起動を伴う通常更新を受ける。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - Hotpatch登録済みのWindows Server 2025ではKB5070881を展開せず、KB5070893を適用する。
    - KB5070881をダウンロード済みなら、Windows Updateを一時停止→再開→再スキャンしてKB5070893を取得する。
    - CISAの指示どおり、CVE-2025-59287の修正を速やかに適用しシステムを保護する。
- その他
    - 脆弱性に関する記事：CVE-2025-59287は既に悪用確認済みで、PoCも公開されている。
    - 攻撃者情報：記載なし。
    - 影響範囲：主にHotpatch登録済みのWindows Server 2025。
- ChatGPTの推奨事項
    - Hotpatch登録済みサーバの更新計画を即時見直し、KB5070881の配布を停止してKB5070893へ切替適用する。

#### 悪性VSX拡張「SleepyDuck」、Ethereumを利用してC2サーバを存続
https://thehackernews.com/2025/11/malicious-vsx-extension-sleepyduck-uses.html

- 要約
    - Open VSXで悪性拡張「juan-bianco.solidity-vlang」が発見。10/31公開のv0.0.7は無害だが、11/1のv0.0.8でRAT「SleepyDuck」を内蔵。
    - 拡張はエディタ起動や.sol選択で発火し、最速のEthereum RPCへ接続してコントラクトを取得、C2情報を動的に読み取り30秒ごとに命令をポーリング。
    - C2はsleepyduck[.]xyzに設定され、コントラクトで切替可能。ホスト名やユーザー名等を収集し送信、緊急指令の一斉配信機能も備える。
    - さらにVS Code市場の「developmentinc」名義で公開された5拡張を確認。ポケモン拡張はmock1[.]su経由でバッチとMoneroマイナーを取得。
    - 利用者には信頼できる発行元のみからの導入と注意を喚起。削除済み拡張はGitHubのRemovedPackages一覧で追跡可能。
- IOCの列挙
    - sleepyduck[.]xyz,FQDN名,知られていない,SleepyDuckのC2ドメインとして拡張が契約から取得し接続,NA(IP以外)
    - mock1[.]su,FQDN名,知られていない,悪性拡張がバッチとMoneroマイナーをダウンロードする外部サーバ(:443),NA(IP以外)
- 推奨事項
    - 拡張は信頼できる発行元からのみ入手・導入する。
    - 不審な拡張のインストールを避け、導入前に慎重に確認する。
    - 削除済み拡張はGitHubのRemovedPackages一覧で確認する。
- その他
    - 脆弱性ではなく、悪性拡張（RAT/マイナー）に関する実害事案。当該拡張は既にダウンロード不可。
    - 攻撃者の属性は不明。目的は暗号資産窃取やマイニングとみられる。
- ChatGPTの推奨事項
    - VS Code/Open VSX環境で当該拡張（juan-bianco.solidity-vlangやdevelopmentinc名義）の有無を今すぐ確認し、削除と端末フルスキャン・資格情報変更を実施。
