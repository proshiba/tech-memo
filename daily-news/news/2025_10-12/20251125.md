#### ClickFix攻撃、偽のWindows Update画面でマルウェアを配布
https://www.bleepingcomputer.com/news/security/clickfix-attack-uses-fake-windows-update-screen-to-push-malware/

- 要約
    - ClickFixの新手口は、全画面の偽「Windows更新」アニメで利用者を誘導し、画像に隠したコードで最終ペイロードを復号・実行する。
    - 被害者は指示どおりのキー操作でコマンドを貼り付け・実行させられ、サイト上のJavaScriptがクリップボードから命令を注入。
    - 解析ではLummaC2とRhadamanthysの情報窃取型が投下され、PNGステガノ＋.NET製Stego Loader＋PowerShell＋mshtaで多段化。
    - Stego LoaderにはAES暗号PNGとDonutが使われ、空関数1万回呼びの回避（ctrampoline）で解析を妨害する工夫が確認された。
    - 10月1日以降に観測。11月13日のOperation Endgameで一部インフラ停止も、偽アップデート用ドメイン自体は稼働中とされる。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - Windowsの［ファイル名を指定して実行］（Runボックス）を無効化すること。
    - explorer.exeからmshta.exeやPowerShellが派生する不審なプロセス連鎖を監視・検知すること。
    - インシデント調査時はRunMRUレジストリキーを確認し、Runボックス入力の痕跡を追跡すること。
- その他
    - なし
- ChatGPTの推奨事項
    - 直ちにRunボックスを無効化し、explorer.exe→mshta.exe/PowerShellのプロセス連鎖検知をEDRで有効化すること。

#### Shai-Huludマルウェアが500のnpmパッケージを感染、GitHubで秘密情報を漏えい
https://www.bleepingcomputer.com/news/security/shai-hulud-malware-infects-500-npm-packages-leaks-secrets-on-github/

- 要約
    - 攻撃者は週末にnpmへ改ざん版を大量投入し、開発者やCI/CDの秘密情報を窃取してGitHub上に自動公開した。
    - Zapier/ENS/PostHog/AsyncAPI等が影響し、約500名称・数万件規模へ拡大、約350のメンテナーアカウント悪用が報告された。
    - 新亜種はsetup_bun.jsと10MBのbun_environment.jsを用い、極度の難読化と多段処理でSecretの抽出と破壊行為を実装。
    - GitHubでは自動生成リポジトリにcloud.json等4ファイルが作成され、記述に「Shai-Hulud: The Second Coming」が付与された。
    - 研究者は感染パッケージの特定と置換、資格情報やトークンの即時ローテーション、CIでpostinstall無効化を勧告している。

    - npm/GitHub/クラウド関連の資格情報・トークンを直ちにローテーションすること。
    - 可能であればCI環境でnpmのpostinstallスクリプトを無効化すること。

#### Microsoft、Windows Server 2025以降でWINSサポートを削除へ
https://www.bleepingcomputer.com/news/microsoft/microsoft-to-remove-wins-support-after-windows-server-2025/

- 要約
    - Microsoftは、Windows Server 2025を最後にWINSサポートを廃止し、以降のサーバー版から機能を削除すると発表した。
    - WINSは2021年8月のWindows Server 2022で非推奨化済みで、2025版の標準サポートは2034年11月まで継続。
    - 削除対象はWINSサーバーロール、管理コンソールのスナップイン、WINS自動化APIなど関連インターフェース一式。
    - 理由はDNSの拡張性と標準準拠、DNSSECによるキャッシュポイズニング/なりすまし防御など、WINS/NetBIOSでは不足するため。
    - 移行先としてDNSの条件付きフォワーダー、スプリットブレインDNS、検索サフィックスの活用を推奨し、依存調査と計画を直ちに開始すべきと助言。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - WINS/NetBIOS名称解決に依存するサービス・アプリの即時監査を実施すること。
    - DNSへの移行（条件付きフォワーダー、スプリットブレインDNS、検索サフィックス）を計画・実装すること。
    - 一時対応としての静的hostsファイルの使用は避けること。
    推奨事項が１つもない場合は、推奨事項なしと記載
- その他
    - なし
- ChatGPTの推奨事項
    - 直ちにWINS依存の資産を棚卸しし、DNS移行計画（条件付きフォワーダー等）を作成して実施に着手すること。

#### Dropping Elephantハッカー集団、MSBuildドロッパー経由のPythonバックドアで防衛セクターを攻撃
https://cybersecuritynews.com/dropping-elephant-hacker-group-attacks-defense-sector/

- 要約
    - インド系とされるDropping Elephantが、パキスタン防衛分野を狙いMSBuildドロッパーでPython系RATを展開する多段攻撃を実施。
    - 偽装の防衛関連フィッシングメールにZIPを同梱し、MSBuildプロジェクトとおとりPDFで初期侵入しタスクスケジューラで永続化。
    - 文字列のUTF逆順化や動的API解決など高度な難読化を用い、KeyboardDrivers/MsEdgeDrivers等の偽名タスクで秘匿化。
    - AppData直下に埋め込みPythonランタイムを配置し、pythonw.exeと偽DLL「python2_pycache_.dll」でバックドア機能を常駐化。
    - C2はnexnxky[.]info等のFQDNを使用。監視強化（MSBuild実行・異常なPython展開・フィッシング対策）が推奨とされる。
- IOCの列挙
    - nexnxky[.]info,FQDN名,知られていない,C2ドメインとして使用,NA(IP以外)
    - upxvion[.]info,FQDN名,知られていない,C2ドメインとして使用,NA(IP以外)
    - soptr[.]info,FQDN名,知られていない,C2ドメインとして使用,NA(IP以外)
- 推奨事項
    - 疑わしいMSBuild（msbuild.exe）実行の監視・検出強化を行うこと。
    - システムディレクトリ/AppData配下への不審なPythonランタイム展開を監視すること。
    - フィッシング対策を強化し、ZIP添付とMSBuildプロジェクト実行を制限すること。
- その他
    - 攻撃者はDropping Elephant（インド系アライメント）。目的はサイバースパイ活動とみられる。
- ChatGPTの推奨事項
