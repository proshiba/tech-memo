#### 暗号資産ウォレット、2025年に不正資金1,580億ドルで過去最多
https://www.bleepingcomputer.com/news/security/crypto-wallets-received-a-record-158-billion-in-illicit-funds-last-year/

- 要約
    - TRM Labs報告では2025年の不正暗号資産フローが1,580億ドルで過去最多。2021年860億→2024年640億の減少傾向を反転。
    - 一方でオンチェーン全体に占める不正割合は2024年1.3%→2025年1.2%に微減し、取引ボリューム拡大が背景にあると示唆。
    - 急増要因は制裁関連活動の拡大（ロシア関連A7/A7A5等）、国家・準国家（ロシア・イラン・ベネズエラ）の中国に関連する地下銀行を使った恒常的利用。
    - ハッキングは150件で被害28.7億ドル、上位10件が81%を占有。最大は2025年2月のBybit侵害で約14.6億ドル、北朝鮮関与とされる。
    - 詐欺は350億ドル送金・投資詐欺が62%。ランサムは高止まりだが支払い忌避が増加、資金洗浄の方法に変化があり、ミキサー利用が-37%で、ブリッジ・クロスチェーンルーティング利用が+66%。

#### Microsoft、将来のWindowsリリースでNTLMを既定で無効化へ
https://www.bleepingcomputer.com/news/microsoft/microsoft-to-disable-ntlm-by-default-in-future-windows-releases/

- 要約
    - Microsoftは、脆弱性悪用リスクの高いNTLM認証を将来のWindowsで既定無効化すると発表。次期Serverと対応クライアントで実施。
    - NTLMは1993年導入。Kerberosが既定だがフォールバックで残り、リレー攻撃やパスザハッシュなどに長年悪用されてきた。
    - 移行は三段階：
        - step1: Windows 11 24H2/Windows Server 2025の強化監査でNTLM使用箇所を洗い出す。
        - step2: 2026年後半にIAKerbとLocal KDCを導入し、Kerberos利用を容易化してNTLMフォールバック要因を解消。
        - step3: 将来リリースでネットワークNTLMを既定無効化。OSには残るが、必要時はポリシーで再有効化できる。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 開発者はNTLMからKerberosまたはNegotiate認証へ移行するようMicrosoftが助言（2024年7月に正式非推奨）。
    - 管理者はNTLMを無効化、またはAD CS構成でNTLMリレー攻撃をブロックするよう推奨。
    - Windows 11 24H2/Server 2025の監査機能で自組織のNTLM利用状況を特定。
- その他
    - なし
- ChatGPTの推奨事項

#### Hugging Faceが悪用され、数千のAndroidマルウェア亜種が拡散
https://www.bleepingcomputer.com/news/security/hugging-face-abused-to-spread-thousands-of-android-malware-variants/

- 要約
    - ルーマニアのBitdefenderが、Hugging Faceを配布基盤に悪用するAndroidキャンペーンを確認し、数千のAPK亜種が展開と報告。
    - 被害者は広告経由で偽セキュリティアプリ「TrustBastion」を導入し、直後にGoogle Playを模した強制更新画面を表示される。
    - ドロッパーはtrustbastion[.]comへ接続し、Hugging Faceのデータセットへリダイレクト、CDN経由で最終ペイロードを取得。

#### Ivanti、ゼロデイ攻撃で悪用されたEPMMの2件の脆弱性を警告
https://www.bleepingcomputer.com/news/security/ivanti-warns-of-two-epmm-flaws-exploited-in-zero-day-attacks/

- 要約
    - Ivanti EPMMにCVE-2026-1281/1340の重大欠陥。認証不要のコード注入でRCEが可能、CVSSはともに9.8。
    - 限定的ながら実際に悪用を確認。CISAは1281をKEVに追加し、米連邦機関へ2026/02/01までの対処を指示。
    - Ivantiは影響バージョン向けにRPMホットフィックスを提供。停止不要・機能影響なしだが、アップグレード後は再適用が必要。
    - 恒久修正はEPMM 12.8.0.0でQ1 2026に提供予定。侵害時はシステムのクリーニングではなく、既知良バックアップからの復元や再構築を推奨。
    - 悪用はIn-House配布/Android File Transfer経由。Apacheアクセスログの特定404を抽出する正規表現で検知可能。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 影響バージョンにIvanti提供のRPMホットフィックスを直ちに適用（停止不要）。恒久修正提供までアップグレード時は再適用。
    - 侵害疑い時はクリーンアップせず、既知良バックアップから復元/再構築し、EPMM・LDAP/KDC等のアカウントのパスワードを全更新。
    - 公開証明書の失効・再発行、Sentryログ/外部保全ログの確認、および提示の正規表現でアクセスログを横断検索。

https://thehackernews.com/2026/01/china-linked-uat-8099-targets-iis.html

https://www.crowdstrike.com/en-us/blog/labyrinth-chollima-evolves-into-three-adversaries/

- 要約
    - CrowdStrikeはLABYRINTH CHOLLIMAを再評価し、GOLDEN・PRESSURE・COREの3組織として追跡する方針を公表。
