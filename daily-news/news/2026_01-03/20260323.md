#### VoidStealerマルウェア、デバッガ手法でChromeのマスターキーを窃取
https://www.bleepingcomputer.com/news/security/voidstealer-malware-steals-chrome-master-key-via-debugger-trick/

- 要約
    - Gen Digitalによると、VoidStealerは実環境で初めて確認された、ハードウェアブレークポイントを使う新しいABE回避手法を備えたインフォスティーラーである。
    - Chrome 127で導入されたApplication-Bound Encryptionは、保存された機微データの復号鍵をディスク上で保護するが、VoidStealerはこの仕組みを回避する。
    - このマルウェアは停止状態の隠しブラウザを起動してデバッガとして接続し、chrome.dllやmsedge.dllの特定命令位置にブレークポイントを設定する。
    - その後、起動時にブラウザが保護データを復号する瞬間を狙い、平文でメモリ上に現れるv20_master_keyをReadProcessMemoryで抜き出す。
    - VoidStealerは少なくとも2025年12月中旬以降ダークウェブでMaaSとして宣伝されており、この技術はChromeKatzのElevationKatzを流用した可能性が高い。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 推奨事項なし
- その他
    - ChromeKatzはOSS: https://github.com/Meckazin/ChromeKatz
- ChatGPTの推奨事項
    - Chrome/Edge利用端末でインフォスティーラー感染の有無を直ちに確認し、保存済み認証情報の失効とOS・ブラウザの更新を実施してください。

#### Trivy脆弱性スキャナー侵害、GitHub Actions経由でインフォスティーラーを配布
https://www.bleepingcomputer.com/news/security/trivy-vulnerability-scanner-breach-pushed-infostealer-via-github-actions/

- 要約
    - Trivyのビルド基盤がサプライチェーン攻撃で侵害され、Trivy v0.69.4とtrivy-action、setup-trivyなどのGitHub Actions経由で認証情報窃取マルウェアが配布された。
    - 攻撃者は、2026-03-01の先行侵害で流出し封じ込めが不十分だった書き込み権限付き認証情報を悪用し、trivy-actionの76個中75個のタグを悪性コミットへ付け替えた。
    - マルウェアはSSH鍵、AWS/GCP/AzureやKubernetes、Dockerの認証情報、.env、各種DB設定、Slack/Discordトークン、/etc/passwdや/etc/shadowなどを収集した。
    - 収集データはtpcp.tar.gzとして暗号化され、タイポスクワットのC2サーバーscan[.]aquasecurtiy[.]orgへ送信され、失敗時は被害者GitHubアカウントの公開リポジトリへ保存された。
    - Aqua Securityは、影響バージョンを使った組織は環境を全面侵害として扱うよう促し、同じ攻撃者によるnpm向け自己増殖ワーム「CanisterWorm」も研究者が関連付けている。
- IOCの列挙
    - scan[.]aquasecurtiy[.]org,FQDN名,知られていない,窃取データの送信先として使われたタイポスクワットのC2サーバー,NA
- 推奨事項
    - 影響を受けたバージョンやタグを使用した組織は、自組織の環境を全面侵害されたものとして扱うこと。
    - クラウド認証情報、SSH鍵、APIトークン、データベースパスワードなど、すべてのシークレットをローテーションすること。
    - システムを分析し、追加の侵害がないか確認すること。
- その他
    - 攻撃者はTeamPCPとの関連が示されており、DeadCatx3、PCPcat、ShellForceとしても追跡され、Docker APIやKubernetes、Ray、Redisの誤設定悪用で知られるクラウドネイティブ脅威アクターと説明されている。
- ChatGPTの推奨事項
    - 影響期間中にTrivy v0.69.4または改ざんタグのtrivy-action/setup-trivyを実行した環境は直ちに隔離し、全シークレットの失効・再発行とGitHub Actionsランナーの持続化痕跡調査を実施してください。

#### Google、Androidで安全なAPKサイドロード向け「Advanced Flow」を追加
https://www.bleepingcomputer.com/news/security/google-adds-advanced-flow-for-safe-apk-sideloading-on-android/

- 要約
    - Googleは、未検証の開発元によるAPKをAndroidへより安全にサイドロードできる新仕組み「Advanced Flow」を発表し、2026年8月に導入予定としている。
    - この仕組みでは、利用者は開発者モードの有効化、攻撃者に誘導されていない確認、端末再起動と再認証、1日待機後の再確認を一度だけ実施する必要がある。
    - その後は未検証の開発元アプリを1週間または無期限で有効化できるが、Androidはそのアプリが未検証の開発元からのものであると警告表示する。
    - Googleは、この追加の手間によって、詐欺師が恐怖や緊急性を利用して利用者に安全機能の無効化や危険なアプリ導入を迫る典型的な手口を妨害できると説明した。
    - さらにGoogleは、配布方法を問わず全Androidアプリ開発者に本人確認を求める開発者認証制度を2026年8月に導入予定で、開発者へ詳細確認を促している。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - アプリ開発者は、2026年8月に予定される開発者認証制度に備え、Googleが案内する詳細情報を確認すること。
- その他
    - なし
- ChatGPTの推奨事項
    - 組織管理端末では直ちに不明な提供元からのAPK導入可否を見直し、業務端末での無許可サイドロードを制限してください。

#### Microsoft Azure Monitorのアラートがコールバック型フィッシング攻撃に悪用
https://www.bleepingcomputer.com/news/security/microsoft-azure-monitor-alerts-abused-in-callback-phishing-campaigns/

- 要約
    - Microsoft Azure Monitorのアラート機能が悪用され、Microsoft Security Teamを装って未承認請求を警告し、記載電話番号へ連絡させるコールバック型フィッシングが確認された。
    - メールは偽装ではなく正規のAzure Monitor基盤から送信されるため、SPF・DKIM・DMARCを通過し、信頼できるMicrosoft通知のように見えてしまう。
    - 攻撃者は、注文・支払い・請求書生成など容易に発火する条件でアラートを作成し、説明欄へ偽の請求メッセージと電話番号を埋め込んで受信者を誘導していた。
    - さらに、攻撃者配下とみられるメーリングリストへ送った後に多数の標的へ転送することで、元のMicrosoftヘッダーや認証結果を保ったまま拡散していた。
    - 記事では、この手口が企業向け通知を装って社内ネットワークへの初期侵入に使われる可能性に触れ、電話番号付きや緊急請求対応を求める通知への警戒を促している。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 電話番号が記載されているAzureまたはMicrosoftのアラートや、請求問題への緊急対応を求める通知は不審なものとして扱うこと。
- その他
    - なし
- ChatGPTの推奨事項
    - Azure/Microsoft名義で電話連絡を求める請求通知は直ちに隔離し、社内ヘルプデスクまたは公式ポータルで真正性を確認してください。

#### FBI、Signalへのフィッシング攻撃をロシア情報機関に関連付け
https://www.bleepingcomputer.com/news/security/fbi-links-signal-phishing-attacks-to-russian-intelligence-services/

- 要約
    - FBIは、SignalやWhatsAppなどの暗号化メッセージングアプリ利用者を狙うフィッシング攻撃について、ロシア情報機関に関連する脅威アクターの活動だと公表した。
    - この攻撃は暗号自体を破るのではなく、アカウント乗っ取りや攻撃者端末の紐付けによって、エンドツーエンド暗号化の保護を回避する手口と説明されている。
    - 被害は世界で数千件規模に及び、現在および元米政府職員、軍関係者、政治家、記者など、機微情報へアクセスできる人物が主な標的とされる。
    - 攻撃では、サポート担当を装うメッセージで認証コードの共有や悪性QRコードの読み取りを誘導し、乗っ取ったアカウントを使ってさらに別の被害者を狙う。
    - FBIは、SignalやWhatsAppの暗号化機能や脆弱性が破られたわけではないとしつつ、不審メッセージや端末連携要求への警戒を呼びかけている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 予期しないメッセージには常に疑いを持つこと。
    - QRコードの読み取りや、自分のアカウントへの端末リンク要求には注意すること。
    - サポート担当を名乗る相手を含め、認証コードを誰にも共有しないこと。
- その他
    - 脆弱性に関する記事ではなく、既に世界で数千件の不正アクセスが発生しているフィッシング攻撃に関する記事である。
    - 攻撃者はロシア情報機関に関連するとされ、機微情報を持つ高情報価値の人物を主な標的としている。
- ChatGPTの推奨事項
    - SignalやWhatsApp利用者は直ちにリンク済み端末を確認し、不審な端末を削除したうえで認証コード共有禁止を周知してください。
