#### バックドア化された Telnyx の PyPI パッケージが、WAV 音声に隠したマルウェアを配布
https://www.bleepingcomputer.com/news/security/backdoored-telnyx-pypi-package-pushes-malware-hidden-in-wav-audio/

- 要約
    - TeamPCP は PyPI 上の Telnyx パッケージを侵害し、WAV ファイルに隠した認証情報窃取マルウェアを含む 4.87.1 と 4.87.2 を公開した。
    - Aikido、Socket、Endor Labs は、過去事案と同じ流出パターンと RSA 鍵に基づき、このサプライチェーン攻撃を TeamPCP に帰属した。
    - Linux/macOS では import 時に telnyx/_client.py が実行され、C2 から ringtone.wav を取得し、埋め込まれた第2段階をメモリ上で動かす。
    - Windows では hangup.wav から msbuild.exe を展開してスタートアップへ置き、Kubernetes があればシークレット列挙や特権 Pod 展開も試みる。
    - 研究者は 4.87.0 へのロールバックを推奨し、悪性版を取り込んだシステムは完全侵害済みとして扱い、全シークレットの即時ローテーションを求めた。
- IOCの列挙
    - 83[.]142[.]209[.]203,IPアドレス,知られていない,C2サーバのIPアドレス,NA(IP以外)
- 推奨事項
    - Telnyx 4.87.1 または 4.87.2 が見つかった場合は、正常な 4.87.0 へロールバックする。
    - 悪性版を import したシステムは、実行時に機密情報が流出済みの可能性を踏まえ、完全に侵害されたものとして扱う。
    - 影響を受けた場合は、すべてのシークレットをできるだけ早くローテーションする。
- その他
    - 攻撃者は TeamPCP とされ、過去事案と同じ流出パターンと RSA 鍵を根拠に帰属されている。
- ChatGPTの推奨事項
    - Telnyx 4.87.1/4.87.2 を直ちに削除して 4.87.0 へ戻し、該当環境を侵害済み前提で調査しつつ全シークレットを即時ローテーションしてください。

#### GitHub上の偽VS Code警告が開発者にマルウェアを拡散
https://www.bleepingcomputer.com/news/security/fake-vs-code-alerts-on-github-spread-malware-to-developers/

- 要約
    - Disscussionsに偽のVS Code脆弱性警告を大量投稿し、開発者にマルウェアをダウンロードさせる大規模攻撃が進行している。
    - 投稿は「重大な脆弱性」など現実味のある題名、偽のCVE番号、緊急性の高い文言を使い、保守担当者や研究者になりすまして信頼性を装う。
    - 攻撃は新規または低活動アカウントから数千のリポジトリへ自動投稿され、通知メールを通じて多数の開発者の受信箱にも直接届く。
    - 偽投稿はGoogle Driveなどの外部配布先へ誘導し、その後 drnatashachinn[.]com にリダイレクトして端末情報を集めるJavaScriptを実行する。
    - 利用者には、NVD・CISA・MITREで脆弱性識別子を確認し、外部ダウンロードリンクや検証不能なCVE、大量タグ付けを詐欺の兆候として見るよう勧告された。
- IOCの列挙
    - drnatashachinn[.]com,FQDN名,知られていない,外部配布リンクからのリダイレクト先として被害者の端末情報を収集するJavaScriptを実行するサイト,NA(IP以外)
- 推奨事項
    - セキュリティ警告を受けたら、NVD、CISA の Known Exploited Vulnerabilities カタログ、MITRE の CVE サイトなど権威ある情報源で脆弱性識別子を検証する。
    - 慌てて行動する前に、通知の正当性を一度立ち止まって確認する。
    - 外部ダウンロードリンク、検証不能なCVE、大量の無関係ユーザーへのタグ付けといった詐欺の兆候を確認する。
- その他
    - 第2段階ペイロードは取得されておらず、少なくとも記事時点ではJavaScript段階での認証情報窃取は確認されていない。
- ChatGPTの推奨事項
    - GitHub Discussions 由来の VS Code 警告メールに含まれる外部配布リンクを直ちに遮断・周知し、drnatashachinn[.]com への通信をブロックしてください。

#### 欧州委員会、Amazon クラウドアカウント侵害後の情報漏えいを調査
https://www.bleepingcomputer.com/news/security/european-commission-investigating-breach-after-amazon-cloud-account-hack/

- 要約
    - 欧州委員会は、攻撃者が同委員会の Amazon クラウド環境にアクセスしたことを受けて、少なくとも1件の AWS アカウントに影響した侵害を調査している。
    - 事件は公表時点で欧州委員会から正式開示されていないが、攻撃は迅速に検知され、同委員会の CSIRT が調査を進めていると報じられた。
    - 攻撃者は BleepingComputer に対し、複数のデータベースを含む 350GB超のデータを盗んだと主張し、委員会職員の情報やメールサーバーへのアクセス証拠を示した。
    - 攻撃者は侵入方法を明かしておらず、盗んだとするデータを用いた恐喝は行わない一方、後日オンラインで公開する意向を示した。
    - Amazon は自社側でセキュリティインシデントは発生しておらず、サービスは想定どおりに動作していたとコメントした。

#### 反著作権侵害連合、ユーザー500万人の AnimePlay アプリを停止
https://www.bleepingcomputer.com/news/security/anti-piracy-coalition-takes-down-animeplay-app-with-5-million-users/

- 要約
    - ACEは、登録ユーザー500万人超・60TB超のアニメ作品を抱える違法配信サービス「AnimePlay」を停止したと発表した。
    - ACEはDisney、Paramount、Sony Pictures、Warner Bros、Netflixなど50超の放送網・映画スタジオに支援され、違法配信の摘発を進めている。
    - 今回の措置でACEはアプリ本体に加え、関連15ドメイン、ホスティング環境、ソースコードなど基盤全体を掌握し、オフライン化した。
    - 運営者はバックエンドサーバー、関連データベース、広告ツール、完全なソースコードを含む29件のGitHubリポジトリの管理権も引き渡した。
