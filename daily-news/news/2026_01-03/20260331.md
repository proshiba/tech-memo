#### 新たなRoadK1ll WebSocketインプラント、侵害済みネットワークでのピボットに使用
https://www.bleepingcomputer.com/news/security/new-roadk1ll-websocket-implant-used-to-pivot-on-breached-networks/

- 要約
    - Blackpointはインシデント対応の過程で、新たに確認したNode.js製インプラント「RoadK1ll」を発見し、侵害済み端末を攻撃者の中継点に変えると説明している。
    - RoadK1llは攻撃者管理下の基盤へ外向きWebSocket接続を張り、その単一トンネル経由でTCP通信を中継して、継続的なアクセスと追加操作を可能にする。
    - 受信待ちリスナーに依存せず、被害端末のネットワーク上の信頼や配置を引き継ぐため、外部公開されていない内部サービスや管理画面にも到達できる。
    - 実装されている主要コマンドは CONNECT、DATA、CONNECTED、CLOSE、ERROR の少数に絞られ、同じトンネル上で複数接続を同時に扱える。
    - レジストリやサービス等の従来的な永続化はないが、通信断時の再接続機能を備え、Blackpointはハッシュと通信元IPを含むIOCがあると述べている。

#### AppleがClickFix攻撃を防ぐためmacOS Terminalに警告を追加
https://www.bleepingcomputer.com/news/security/apple-adds-macos-terminal-warning-to-block-clickfix-attacks/

- 要約
    - AppleはmacOS Tahoe 26.4で、Terminalに貼り付けた危険な可能性のあるコマンドの実行を一時停止し、警告を出す新機能を導入した。
    - この仕組みは主にClickFix対策を狙ったものとみられ、被害者に問題解決や認証確認を装って悪性コマンドを貼り付けさせる手口を抑止する。
    - 警告では、システムにまだ被害は出ておらず、詐欺師がさまざまな経路で悪意ある指示を配布していることが利用者へ説明される。
    - 利用者は貼り付けを中止することも無視して続行することもできるが、後者はコマンドの影響を理解している場合に限るべきとされる。
    - ただしAppleは公式サポート文書を公開しておらず、どのように危険判定しているか不明なため、新警告だけに全面的に頼るべきではない。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - オンラインで見つけたコマンドの内容を十分に理解していない場合は、実行しない。
    - 警告が表示され、指示元が信頼できない、またはコマンドの意味が分からない場合は貼り付けを中止する。
    - macOSの新しい警告機能だけに全面的に依存しない。
- その他
    - 脆弱性の発見や修正情報を扱う記事ではなく、ClickFix攻撃を抑止するためのmacOS側の新しい防御機能について述べた内容である。
    - 攻撃者の国籍や動機などの属性情報は記事中に記載がなく、不明。
- ChatGPTの推奨事項
    - macOS利用者には、SafariなどからTerminalへコマンドを貼り付けて実行する運用を今すぐ見直し、注意喚起を周知してください。

#### 攻撃者が重大なF5 BIG-IPの欠陥を攻撃で悪用中、今すぐパッチを適用
https://www.bleepingcomputer.com/news/security/hackers-now-exploit-critical-f5-big-ip-flaw-in-attacks-patch-now/

- 要約
    - F5は、従来DoSとして扱っていたBIG-IP APMの脆弱性CVE-2025-53521を、未修正機器にWebシェルを設置可能な重大なRCEとして再分類した。
    - この欠陥は、仮想サーバー上でアクセスポリシーが構成されたBIG-IP APMを対象に、権限のない攻撃者でもリモートコード実行を起こせるとされる。
    - F5は2026年3月に得た新情報をもとに、既存の修正版がRCEにも有効であると再確認しつつ、実際に脆弱バージョンで悪用が発生していると警告した。
    - さらにCISAはこの脆弱性を既知の悪用済み脆弱性一覧へ追加し、米連邦機関に対し2026年3月30日までの保護対応を命じた。
    - Shadowserverはインターネット上に24万超のBIG-IPが露出していると追跡しているが、脆弱な構成数や既に保護済みの台数は不明とされる。

- 推奨事項
    - BIG-IPシステムのディスク、ログ、ターミナル履歴を確認し、悪意ある活動の痕跡がないか調査する。
    - 復旧を試みる前に、自組織のインシデント対応方針と証拠保全・フォレンジック手順を確認する。
    - ベンダーの指示に従って緩和策や修正を適用し、緩和できない場合は製品利用の停止も検討する。

#### 重大なFortinet FortiClient EMSの脆弱性、攻撃で悪用中
https://www.bleepingcomputer.com/news/security/critical-fortinet-forticlient-ems-flaw-now-exploited-in-attacks/

- 要約
    - Defusedによると、FortinetのFortiClient EMSにある重大SQLインジェクション脆弱性CVE-2026-21643が、すでに実際の攻撃で悪用されている。
    - この欠陥はFortiClient EMSのGUI（Web管理画面）に細工したHTTPリクエストを送ることで、未認証の攻撃者でも低難度で任意コードや任意コマンド実行を狙える。
    - 攻撃ではHTTPリクエスト内の「Site」ヘッダーを通じてSQL文を忍ばせられるとされ、Shodan上では公開されたFortiClient EMSが約1,000件近くあると説明されている。
    - Fortinetの製品セキュリティチームが社内で発見した問題で、影響を受けるのは7.4.4であり、7.4.5以降へ更新することで修正できる。
    - ShadowserverはWeb管理画面が公開されたFortiClient EMSを2,000超追跡しており、米国や欧州に多い。
- IOCの列挙

- 推奨事項
    - 影響を受けるFortiClient EMS 7.4.4を、7.4.5以降へアップグレードする。

#### CTRLの正体：これまで文書化されていなかったロシア製 .NET アクセス・フレームワークの解剖
https://censys.com/blog/under-ctrl-dissecting-a-previously-undocumented-russian-net-access-framework/

- 要約
    - Censys ARCは、LNKファイル1本から展開される未文書化のロシア系 .NET 製RAT「CTRL」を確認し、公開ディレクトリと稼働中インフラから分析した。
    - このツールキットは、偽装LNK→多層PowerShell→レジストリ常駐ステージャ→ctrl[.]exe/FRP/RDP Wrapperという6段階で感染を進める。
    - 機能面では、Windows Hello風PINフィッシング、キー入力記録、RDP乗っ取り、FRPリバーストンネル、通知なりすまし、隠し管理者作成を備える。
    - C2は直接ビーコンせず、hui228[.]ru を介したFRPトンネルとRDP操作に寄せる設計で、公開脅威情報やVirusTotalにも当時未登録だった。
    - 観測された中継基盤は 194.33.61[.]36 と 109.107.168[.]18 で、著者は新規調達またはローテーション中の可能性を示している。

    - hui228[.]ru:82/hosted/,URL,知られていない,3つの .NET 実行ファイルを格納した公開ペイロードディレクトリとして紹介,NA(IP以外)
    - 194.33.61[.]36,IPアドレス,知られていない,hui228[.]ru に紐づくペイロード配布およびFRPリレーの観測先として紹介,グローバル
    - 109.107.168[.]18,IPアドレス,知られていない,2026年2月27日にDNSが切り替わったFRPリレー先として紹介,グローバル
    - 146.19.213[.]155,IPアドレス,知られていない,悪性LNKを含む公開ディレクトリが見つかった別サーバーとして紹介,グローバル
- 推奨事項
    - HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\ 配下への不審なバイナリ書き込み、`DriverSvcTask` などの予定タスク、`C:\Temp\keylog.txt` を監視する。
    - `194.33.61[.]36` と `109.107.168[.]18` の 7000/TCP への外向き通信、FRPプロトコル、908/TCP の自己署名証明書 `CN=DESKTOP-GI428R8` を検知・遮断する。
    - 2つのIPとSSHホスト鍵フィンガープリントを監視対象に加え、PARTNER-HOSTING-LTD ASN 内の FRPS サービスを継続監視する。

#### DeepLoadマルウェア、ClickFix配信とAI生成の回避手法を組み合わせる
https://reliaquest.com/blog/threat-spotlight-deepload-malware-pairs-clickfix-delivery-with-ai-generated-evasion/

- 要約
    - ReliaQuestは、DeepLoadが企業環境で悪用されており、ClickFixで利用者にコマンドを実行させて短時間で永続化と資格情報窃取を成立させると報告している。
    - 初期段階では `mshta.exe` と難読化PowerShellローダーが使われ、AI支援とみられる大量のノイズコードで静的検知を回避しつつ、ペイロードは主にメモリ上で展開される。
    - 本体は `LockAppHost.exe` などの正規WindowsプロセスへAPCインジェクションで潜み、別系統の `filemanager.exe` と悪性ブラウザ拡張で保存済み資格情報やセッションを窃取する。
    - 調査事例ではUSB経由の拡散も見られ、`ChromeSetup.lnk` などのLNKを大量配置して別端末への再感染を狙い、初期感染端末だけが被害とは限らないとされる。
    - さらにWMIイベントサブスクリプションが残ると、通常のクリーンアップ後でも3日後に再実行されるため、記事は挙動監視と資格情報ローテーションの重要性を強調している。
- IOCの列挙
    - holiday-updateservice[.]com,FQDN名,知られていない,DeepLoadのステージングドメインとして紹介,NA(IP以外)
    - forest-entity[.]cc,FQDN名,知られていない,DeepLoadのステージングドメインとして紹介,NA(IP以外)
    - hell1-kitty[.]cc,FQDN名,知られていない,関連する支援インフラとして紹介,NA(IP以外)
- 推奨事項
    - PowerShell Script Block Logging を有効化する。
    - 露出ホストでWMIイベントサブスクリプションを監査し、正当性を確認できない項目を削除する。
    - 感染期間中に到達可能だった保存済みパスワード、セッショントークン、利用中アカウントを含む全資格情報をローテーションする。
