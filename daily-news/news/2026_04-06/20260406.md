#### 交通違反詐欺、新たなフィッシングSMSでQRコードへ移行
https://www.bleepingcomputer.com/news/security/traffic-violation-scams-switch-to-qr-codes-in-new-phishing-texts/

- 要約
    - 米国各州の裁判所を装う詐欺師が、偽の「最終通知」交通違反テキストを送り、QRコード経由でフィッシングサイトへ誘導している。
    - この手口は2025年の通行料・駐車違反詐欺の派生で、今回は本文リンクではなく、裁判所通知画像に埋め込んだQRコードを使う点が新しい。
    - QRコードを読み取ると、まず CAPTCHA を解かせる中継サイトへ送られ、その後 DMV など州機関を装う別の詐欺サイトへ転送される。
    - 記事で確認された例では未払い額は一律 6.99 ドルとされ、氏名・住所・電話番号・メール・クレジットカード情報の入力を求められる。
    - 盗まれた情報は追加のフィッシング、金融詐欺、個人情報窃取、他の脅威アクターへの転売に悪用され得ると説明されている。

    - ny[.]ofkhv[.]life,FQDN名,知られていない,ニューヨーク州 DMV を装うフィッシングサイトのホスト名として紹介,NA
- 推奨事項

#### Axios の npm ハック、偽の Teams エラー修正でメンテナーのアカウントを乗っ取る
https://www.bleepingcomputer.com/news/security/axios-npm-hack-used-fake-teams-error-fix-to-hijack-maintainer-account/

- 要約
    - 人気HTTPクライアント Axios のメンテナーが標的型ソーシャルエンジニアリング被害を受け、npm に悪性版 1.14.1 と 0.30.4 が公開された。
    - 攻撃者は実在企業を装った Slack ワークスペースと Microsoft Teams 会議を用意し、偽の技術エラー修正を口実に RAT を導入させた。
    - 改ざん版は plain-crypto-js 依存関係を注入し、macOS・Windows・Linux に RAT を展開、約3時間公開された後に削除された。
    - 記事によると、当該版を導入したシステムは侵害済みとみなすべきで、資格情報と認証鍵のローテーションが必要とされた。
    - Google Threat Intelligence Group は本件を、北朝鮮系で金銭目的の脅威アクター UNC1069 に結び付けている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 改ざん版を導入した可能性のあるシステムは、侵害済みとして扱う。
    - 影響を受けた資格情報と認証鍵をリセット／ローテーションする。
    - 影響システムをワイプし、同様の事案を防ぐ変更を実施する。
- その他
    - 攻撃者は Google により UNC1069 と関連付けられた、北朝鮮系で金銭目的の脅威アクターと説明されている。

#### 新たなキットの拡散でデバイスコードフィッシング攻撃が37倍に急増
https://www.bleepingcomputer.com/news/security/device-code-phishing-attacks-surge-37x-as-new-kits-spread-online/

- 要約
    - OAuth 2.0 の Device Authorization Grant を悪用してアカウントを奪うデバイスコードフィッシングが、2026年に入って 37.5 倍へ急増した。
    - 攻撃者は取得したデバイス認証コードを被害者に入力させ、正規ログイン画面で承認させることで有効なアクセストークンと更新トークンを得る。
    - Push Security は、EvilTokens を筆頭に少なくとも 11 種のキットがこの手口を提供し、低技能のサイバー犯罪者にも広がっていると説明した。
    - 記事では、VENOM、AUTHOV、DOCUPOLL、FLOW_TOKEN など複数キットが、SaaS を模した誘導文面やクラウド基盤、アンチボット機能を用いると紹介している。
    - 対策として、不要ならデバイスコードフローを条件付きアクセスで無効化し、想定外の認証イベントや不審な IP・セッションの監視が推奨されている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 不要な場合は、アカウントの条件付きアクセス ポリシーでデバイスコードフローを無効化する。
    - 予期しないデバイスコード認証イベント、不審な IP アドレス、異常なセッションをログ監視する。
- その他
    - 本記事は既に悪用が進行している攻撃手法とその拡大を扱っており、発見や修正情報のみの記事ではない。
    - 攻撃者は国家系ハッカーと金銭目的の攻撃者の双方を含み、EvilTokens などにより低技能のサイバー犯罪者にも利用可能になっている。
- ChatGPTの推奨事項
    - Microsoft 365 / Entra などでデバイスコードフローの利用状況を直ちに棚卸しし、不要なら無効化したうえで、直近の認証ログから不審なデバイスコード認証を確認してください。

#### 北朝鮮マルウェアのモジュール化：多様性と機能特化
https://dti.domaintools.com/research/dprk-malware-modularity-diversity-and-functional-specialization

- 要約
    - 本記事は、北朝鮮のサイバー活動を単一集団ではなく、任務別に分業された統制下のマルウェア生態系として分析している。
    - こうした分業化は、国際制裁、法執行の強化、マルウェア公開分析の蓄積で個別ツールの寿命が短くなったことへの合理的な適応だと位置付けている。
    - 北朝鮮のモデルが他の国家主導のAPT（ロシア、中国、イランなど）と決定的に異なる点は、マルウェアを貴重な資産ではなく消耗品として扱っていること。
    - 諜報系は Kimsuky を代表例とし、PowerShell や VBS、ソーシャルエンジニアリング、クラウド悪用、長期潜伏による静かな情報収集を重視すると説明する。
    - 金銭獲得系は Lazarus を中心に、暗号資産事業者や開発者基盤を狙い、ウォレット窃取やサプライチェーン侵害で素早い収益化を図るとされる。
    - 破壊・威圧系は Andariel に結び付けられ、ワイパーや急速な横展開で短時間に目立つ被害を与え、政治的メッセージを伴うと述べる。
    - 防御側には、シグネチャや単発 IOC 偏重を改め、行動分析、ID 監視、サプライチェーン保護、クラウド横断監視を重視すべきだと提言している。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - マルウェアの静的シグネチャだけに依存せず、アクセス、実行、データ移動の異常を捉える行動分析を重視する。
    - 資格情報悪用や信頼済みアカウントの不正利用を前提に、ID とアクセス監視を強化する。
    - 開発者エコシステムとサプライチェーンの保護を強化し、認証イベント、API 利用、サービス間活動を横断して監視する。
- その他
    - 攻撃者は北朝鮮の国家系サイバー活動として説明され、主な動機は諜報収集、制裁下での資金獲得、破壊・威圧である。
- ChatGPTの推奨事項
    - 直ちに認証ログ、PowerShell/VBS 実行、クラウド API 利用、開発者向け供給網のイベントを相関監視できる検知ルールを整備してください。
