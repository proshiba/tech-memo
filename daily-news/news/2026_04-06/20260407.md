#### 不満を抱いた研究者が「BlueHammer」Windowsゼロデイ攻撃コードを公開
https://www.bleepingcomputer.com/news/security/disgruntled-researcher-leaks-bluehammer-windows-zero-day-exploit/

- 要約
    - Microsoft に非公開報告されていた未修正の Windows 権限昇格脆弱性「BlueHammer」の攻撃コードが公開され、SYSTEM または昇格した管理者権限の取得が可能と報じられた。
    - 公開者は Chaotic Eclipse / Nightmare-Eclipse を名乗る研究者で、MSRC の開示対応に強い不満を示し、2026年4月3日に GitHub 上へ PoC を公開した。
    - Will Dormann は、この脆弱性が TOCTOU とパス混同を組み合わせたローカル権限昇格であり、SAM データベースに到達し得ると BleepingComputer に説明した。
    - 攻撃コードには不具合もあり、研究者らの検証では Windows Server で安定して成功しない例があった一方、端末では SYSTEM シェル生成など完全侵害につながり得る。
    - 公開時点で Microsoft の公式パッチや更新はなく、ローカル侵入経路を得た攻撃者に悪用されれば重大な影響を及ぼすゼロデイと位置付けられている。

#### Microsoft、Medusaランサムウェアのアフィリエイトをゼロデイ攻撃と関連付け
https://www.bleepingcomputer.com/news/security/microsoft-links-medusa-ransomware-affiliate-to-zero-day-attacks/

- 要約
    - Microsoft は、Medusa ランサムウェアの展開で知られる中国拠点の金銭目的グループ Storm-1175 が、n-day と zero-day を高速に悪用していると報告した。
    - 同グループは新規脆弱性の武器化が非常に速く、場合によっては公開前の週から悪用し、初期侵入から数日、時に24時間以内に暗号化まで進むとされる。
    - 攻撃では複数の脆弱性を連鎖利用し、新規ユーザー作成、RMM 導入、認証情報窃取、セキュリティ機能停止を経て、ランサムウェア投下とデータ流出に至る。
    - 具体例として、GoAnywhere MFT の CVE-2025-10035 を修正前から1週間超悪用し、SmarterMail の CVE-2026-23760 もゼロデイとして利用したと記載されている。
    - さらに Exchange、PaperCut、Ivanti、ScreenConnect、TeamCity、SimpleHelp、CrushFTP、SmarterMail、BeyondTrust など計10製品超の脆弱性悪用が確認された。
    - Storm-1175は、公開資産の脆弱性を特定する能力に長けており、エクスプロイトブローカーなどを通じて高度な開発能力やリソースへのアクセスを得ている可能性があると分析されている。

#### CISA、悪用中の Fortinet EMS 脆弱性を金曜までに修正するよう連邦機関に命令
https://www.bleepingcomputer.com/news/security/cisa-orders-feds-to-patch-fortinet-flaw-exploited-in-attacks-by-friday/

- 要約
    - CISA は、FortiClient Enterprise Management Server(EMS) の脆弱性 CVE-2026-35616 について、連邦機関に対し今週中の対処を命じた。
    - この欠陥は認証前の API アクセスバイパスで、認証・認可を完全に迂回され、細工したリクエストでコードやコマンド実行に悪用され得る。
    - Fortinet は週末に緊急ホットフィックスを公開し、本脆弱性がゼロデイ攻撃で実際に悪用されているとして、早急な保護を呼びかけた。
    - CISA は 2026年4月6日に本件を KEV に追加し、BOD 22-01 に基づき FCEB 機関へ 2026年4月9日深夜までの修正を要求した。
    - Shadowserver は公開中の FortiClient EMS を約 2,000 件追跡しており、そのうち 1,400 超の IP が米国と欧州にあると記事は伝えている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - FortiClient EMS 7.4.5 および 7.4.6 の利用者は、ベンダー提供の緊急ホットフィックスを適用する。
    - FortiClient EMS 7.4.7 が利用可能になったら、当該バージョンへ更新する。
    - ベンダー指示の緩和策を適用し、クラウドサービスは BOD 22-01 に従い、緩和策がない場合は製品利用を停止する。
- その他

- ChatGPTの推奨事項

#### LNK と GitHub C2 を用いる北朝鮮関連キャンペーン
https://www.fortinet.com/blog/threat-research/dprk-related-campaigns-with-lnk-and-github-c2

- 要約
    - FortiGuard Labs は、韓国の利用者を狙う LNK ファイル群を確認し、GitHub を C2 に悪用する多段階攻撃として分析している。
    - 近年版では、LNK 引数内に復号関数を埋め込み、囮 PDF と次段の PowerShell をファイル内部に符号化して格納していた。
    - 復号後の PowerShell は解析環境を確認し、VBScript と Scheduled Task を使って 30 分ごとの永続実行を確立する。
    - その後、OS 情報、起動時刻、実行中プロセス、ネットワーク構成を収集し、GitHub 上の攻撃者管理先へ送信・追加取得を行う。
    - 記事は、Windows 標準機能と正規の公開基盤を悪用することで検知を避けつつ、韓国企業への監視拡大を狙う活動だと示している。
- IOCの列挙
    - IOCはこちら: https://www.fortinet.com/blog/threat-research/dprk-related-campaigns-with-lnk-and-github-c2
- 推奨事項
    - 信頼できない文書に対して警戒を維持する。
    - 環境内の異常な PowerShell または VBScript の活動を監視する。
- その他
    - 攻撃者は DPRK 関連とされ、北朝鮮国家支援グループに見られる命名や運用上の特徴を示し、韓国企業への監視拡大を狙っていると記載されている。
- ChatGPTの推奨事項
    - Windows 端末で LNK 起点の不審実行、30 分周期の Scheduled Task 作成、wscript.exe と PowerShell の連鎖実行を直ちに点検してください。
