#### 攻撃者、1ピクセル大のSVGトリックでクレジットカード窃取コードを隠蔽
https://www.bleepingcomputer.com/news/security/hackers-use-pixel-large-svg-trick-to-hide-credit-card-stealer/

- 要約
    - Magento を使う約100のオンラインストアに影響する大規模攻撃で、決済情報を盗むコードが 1x1 ピクセルの SVG 画像に埋め込まれていた。
    - Sansec は、攻撃者が 3 月中旬に公表された PolyShell 脆弱性を悪用してサイトへ侵入した可能性が高いとみている。
    - 不正コードは HTML 内の SVG の onload 属性に base64 で隠され、外部スクリプト参照を作らずに実行されるため検知を回避しやすい。
    - 利用者が checkout を押すと偽の Secure Checkout オーバーレイが表示され、カード情報や請求先情報の入力を促し、検証したうえで暗号化して攻撃者に送信される。
    - Adobe は本番版 Magento 向け修正をまだ公開しておらず、修正はプレリリース版 2.4.9-alpha3+ にのみ含まれると記事は述べる。

#### 新たな macOS スティーラー攻撃、ClickFix で Script Editor を悪用
https://www.bleepingcomputer.com/news/security/new-macos-stealer-campaign-uses-script-editor-in-clickfix-attack/

- 要約
    - Jamf は、偽の Apple 風ディスク容量回復ガイドで macOS 利用者を誘導し、Atomic Stealer を配布する新たな ClickFix 亜種を確認した。
    - 従来の Terminal 実行型と異なり、今回は AppleScript 用 URL スキームで Script Editor を開き、実行コードをあらかじめ入力した状態にする。
    - 被害者が Script Editor を許可すると、難読化された curl | zsh がメモリ上でスクリプトを実行し、一時領域のバイナリを取得して起動する。
    - macOSでは、ターミナルでのコマンド実行を伴うClickFix攻撃に対して警告が表示されるようになったが、今回の手法はスクリプトエディタを介することで、ユーザーに手動でコマンドを入力させないことで回避している。
    - 最終ペイロードは AMOS と識別され、Keychain、ブラウザーのパスワードや Cookie、保存カード情報、暗号資産ウォレットなどを狙う。
    - 記事は、Script Editor の実行要求を高リスクとして扱い、macOS のトラブル対応は Apple の公式資料に限定するよう勧めている。

#### 13年前から潜んでいた ActiveMQ のバグで、ハッカーが遠隔でコマンド実行可能に
https://www.bleepingcomputer.com/news/security/13-year-old-bug-in-activemq-lets-hackers-remotely-execute-commands/

- 要約
    - Apache ActiveMQ Classic で、13年間見逃されていた高深刻度の RCE 脆弱性 CVE-2026-34197(CVSS 8.8)が発見され、任意コマンド実行が可能と報じられた。
    - 影響は 5.19.4 未満と 6.0.0 から 6.2.3 までで、Jolokia 管理 API の addNetworkConnector を悪用し、外部 Spring XML を読み込ませて実行に至る。
    - 通常は Jolokia 認証が必要だが、別の欠陥 CVE-2024-32114 の影響で、6.0.0 から 6.1.1 では認証なし RCE になり得ると説明されている。
    - 研究者は Claude を用いて、Jolokia、JMX、network connectors、VM transports など個別機能の組み合わせから攻撃経路を見つけたと述べた。
    - Apache は 2026年3月30日に 6.2.3 と 5.19.4 で修正し、研究者は ActiveMQ は実運用で繰り返し狙われるため高優先で扱うべきだと勧告している。

- 推奨事項
    - ActiveMQ を運用する組織は、本件を高優先度で対処すべきとされている。
    - ブローカーログで、内部転送プロトコル VM を使う不審な broker 接続や、brokerConfig=xbean:hxxp:// クエリパラメータを確認する。
    - 設定問題に関する警告が複数回の接続試行中に出ている場合、既にペイロードが実行された可能性がある点に注意する。
