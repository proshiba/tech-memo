#### 医療ITソリューション提供企業 ChipSoft がランサムウェア攻撃を受ける
https://www.bleepingcomputer.com/news/security/healthcare-it-solutions-provider-chipsoft-hit-by-ransomware-attack/

#### Google Chrome がセッション Cookie 窃取に対するインフォスティーラー対策を追加
https://www.bleepingcomputer.com/news/security/google-chrome-adds-infostealer-protection-against-session-cookie-theft/

- 要約
    - Google は Windows 向け Chrome 146 で Device Bound Session Credentials(DBSC) を導入し、インフォスティーラーによるセッション Cookie 悪用を防ぎ始めた。
    - DBSC はセッションを TPM など端末固有のハードウェア鍵へ暗号学的に結び付け、盗まれた Cookie 単体では再利用できないようにする。
    - 新しい短寿命セッション Cookie の発行は、対応する秘密鍵の所持証明が前提となるため、流出した Cookie はすぐに無効化されやすい。
    - Google は Okta など複数の Web プラットフォームと 1 年間試験し、セッション窃取事案の顕著な減少を確認したと説明している。
    - macOS への対応は将来の Chrome リリースで提供予定で、Web サイト側は登録用・更新用エンドポイント追加で DBSC を実装できる。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - Web サイトは、登録用と更新用の専用エンドポイントをバックエンドへ追加することで、より安全なハードウェアバインド型セッションへ移行できる。
    - Web 開発者は、Google が公開している DBSC の実装ガイドを参照できる。
- その他
    - なし
- ChatGPTの推奨事項
    - Windows 端末の Chrome を直ちに最新化し、社内 Web サービスで DBSC 対応の検証開始可否を確認してください。

#### 攻撃者、12月以降 Acrobat Reader のゼロデイ脆弱性を悪用
https://www.bleepingcomputer.com/news/security/hackers-exploiting-acrobat-reader-zero-day-flaw-since-december/

- 要約
    - 攻撃者は少なくとも2025年12月以降、細工したPDFを使って未修正の Adobe Reader ゼロデイを悪用しており、最新バージョンでも PDF を開くだけで発火するとされた。
    - 発見者の Haifei Li は、この攻撃を高度なフィンガープリンティング型 PDF エクスプロイトと説明し、少なくとも4か月間 Adobe 利用者を標的にしていたと述べた。
    - 攻撃では、権限付きの util.readFileIntoStream と RSS.addFeed API を使って端末内情報を窃取し、追加のエクスプロイトも配備していたという。
    - さらに後続の RCE やサンドボックス回避へ発展し得るため、情報窃取にとどまらず、最終的に被害端末を完全に制御される可能性があると警告された。
    - 解析者は、この攻撃で配布された PDF に、ロシアの石油・ガス業界の進行中の出来事に言及するロシア語の誘導文書が含まれていたと報告した。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 修正プログラムが公開されるまで、信頼できない連絡先から受け取った PDF 文書を開かない。
    - User-Agent ヘッダーに "Adobe Synchronizer" を含む HTTP/HTTPS 通信を監視し、遮断する。
- その他
    - 攻撃者の国籍や組織は不明だが、配布 PDF にはロシア語の誘導文面とロシア石油・ガス業界の話題が使われていた。
- ChatGPTの推奨事項
    - Adobe Reader 利用端末で未信頼 PDF の開封を直ちに停止し、"Adobe Synchronizer" を含む外向き通信の監視・遮断を実施してください。

#### Google Storage 経由の新たなフィッシング攻撃で Remcos RAT を配布
https://cybersecuritynews.com/new-phishing-attack-via-google-storage/

- 要約
    - 新たに確認されたフィッシング攻撃は、Google Cloud Storage 上の悪性 HTML ページを使い、世界中の被害者に Remcos RAT を配布している。
    - 攻撃者は googleapis[.]com 上に Google Drive の共有画面を模したページを置き、信頼された Google 基盤を悪用して検知や遮断を回避する。
    - 被害者がページを開いて操作すると、JavaScript によるリダイレクトや自動ダウンロードが始まり、圧縮または難読化されたアーカイブが取得される。
    - その後、VBScript や PowerShell を使うドロッパーが最終段階の Remcos RAT を取得し、正規プロセスへの hollowing で実行する。
    - 感染後はレジストリを使って永続化され、暗号化された通信で攻撃者サーバーと接続し、キーログ、画面取得、ファイル操作などが可能になる。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 通常業務フローにない googleapis[.]com への外向き通信を監視する。
    - スクリプト実行ポリシーを強制し、振る舞い検知型のエンドポイント検知を有効化し、宛先ドメインに関係なくメール内リンクを検査する。
    - 予期しないメール内リンクを開かないよう利用者教育を行い、共有ファイルを開く前に別経路で送信者を確認する。
- その他
    - なし
- ChatGPTの推奨事項
    - メール経由の googleapis[.]com リンクと VBScript/PowerShell 実行の痕跡を直ちに確認し、該当端末を隔離してレジストリ永続化を点検してください。
