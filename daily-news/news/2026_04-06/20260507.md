#### 重大なvm2サンドボックス脆弱性により攻撃者がホスト上でコード実行可能
https://www.bleepingcomputer.com/news/security/critical-vm2-sandbox-bug-lets-attackers-execute-code-on-hosts/

- 要約
    - Node.jsのサンドボックスライブラリvm2に、ホスト上で任意コード実行可能な重大脆弱性CVE-2026-26956が見つかった。
    - 影響が確認されたのはvm2 3.10.4で、以前のリリースも脆弱な可能性があり、PoCコードも公開されている。
    - 影響条件は、WebAssembly例外処理とJSTagサポートを有効にしたNode.js 25環境と説明されている。
    - 脆弱性は、サンドボックスとホスト間をまたぐ例外処理の不備により、ホスト側エラーオブジェクトが漏れることで発生する。
    - 攻撃者は漏えいしたオブジェクトのconstructorチェーンを悪用し、processなどNode.js内部へアクセスしてコマンド実行できる。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - CVE-2026-26956の悪用リスクを軽減するため、vm2を3.10.5以降へできるだけ早くアップグレードする。
    - 記事時点の最新リリースであるvm2 3.11.2への更新も推奨されている。
- その他
    - 脆弱性に関する記事であり、PoCは公開済みだが、記事内では実際の攻撃で悪用されたとは記載されていない。
    - 一次ソース: https://github.com/patriksimek/vm2/security/advisories/GHSA-ffh4-j6h5-pg66
- ChatGPTの推奨事項
    - vm2 3.10.4の利用有無を今すぐ確認し、該当環境では3.10.5以降へ更新してNode.js 25環境の実行サービスを優先点検してください。

#### 新しいCisco DoS脆弱性、デバイス復旧に手動再起動が必要
https://www.bleepingcomputer.com/news/security/new-cisco-dos-flaw-requires-manual-reboot-to-revive-devices/

- 要約
    - CiscoはCrosswork Network ControllerとNetwork Services OrchestratorのDoS脆弱性修正版を公開した。
    - CVE-2026-20188は受信接続のレート制限不備に起因し、未認証のリモート攻撃者が低複雑度で悪用可能。
    - 悪用されると接続リソースが枯渇し、CNCとNSOが応答不能となり、復旧には手動再起動が必要になる。
    - 影響はCisco CNC 7.1以前とCisco NSO 6.3以前および6.4で、CNC 7.2とNSO 6.5は脆弱ではない。
    - Cisco PSIRTは記事時点で進行中の悪用を把握しておらず、実環境での悪用も未確認とされている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - Ciscoは顧客に対し、アドバイザリで示された修正版ソフトウェアへのアップグレードを強く推奨している。
    - Cisco CNC 7.1以前は修正版リリースへ移行する。
    - Cisco NSO 6.4は6.4.1.3へ更新し、6.3以前は修正版リリースへ移行する。
- その他
    - 脆弱性に関する記事であり、CVE-2026-20188は記事時点で実際の悪用や進行中の攻撃は確認されていない。
    - 一次ソース: https://sec.cloudapps.cisco.com/security/center/content/CiscoSecurityAdvisory/cisco-sa-nso-dos-7Egqyc
- ChatGPTの推奨事項
    - Cisco CNC/NSOのバージョンを今すぐ棚卸しし、影響版は修正版へ更新して外部から到達可能な管理面を制限してください。

#### Palo Alto Networks、攻撃で悪用中のファイアウォールRCEゼロデイを警告
https://www.bleepingcomputer.com/news/security/palo-alto-networks-warns-of-actively-exploited-firewall-zero-day/

- 要約
    - Palo Alto Networksは、PAN-OS User-ID Authentication Portalの重大な未修正脆弱性を警告した。
    - CVE-2026-0300はバッファオーバーフローで、未認証攻撃者がroot権限で任意コード実行できる。
    - 対象はUser-ID Authentication Portalを公開しているPA-SeriesおよびVM-Seriesファイアウォール。
    - Palo Alto Networksは限定的な悪用を確認しており、公開インターネットや信頼できないIPに晒された環境が狙われた。
    - 修正版は作業中で、初回更新は2026年5月13日に提供予定と説明されている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - User-ID Authentication Portalへのアクセスを信頼済みゾーンまたは信頼済み内部ネットワークのみに制限する。
    - それが不可能な場合、User-ID Authentication Portalを無効化する。
    - Device > User Identification > Authentication Portal SettingsでEnable Authentication Portalの有効状態を確認する。
- その他
    - 攻撃者の国や属性、動機は不明で、記事では「攻撃で悪用」とのみ記載されている。
    - 一次ソース: https://security.paloaltonetworks.com/CVE-2026-0300
- ChatGPTの推奨事項
    - 公開中のPAN-OS User-ID Authentication Portalを今すぐ棚卸しし、外部公開を止めて修正版提供後に速やかに適用してください。

#### 新しいステルス型Quasar Linuxマルウェア、ソフトウェア開発者を標的に
https://www.bleepingcomputer.com/news/security/new-stealthy-quasar-linux-malware-targets-software-developers/

- 要約
    - 未文書化のLinuxインプラントQuasar Linux（QLNX）が、開発者端末を標的にしている。
    - QLNXはrootkit、バックドア、認証情報窃取機能を備え、開発・DevOps環境を狙う。
    - gccでrootkit共有オブジェクトやPAMバックドアを動的コンパイルし、長期潜伏を図る。
    - LD_PRELOAD、systemd、crontab、init.d、XDG autostart、.bashrcなど複数の永続化を使う。
    - SSH鍵、ブラウザ、クラウド・開発者設定、/etc/shadow、クリップボードなどを窃取可能。
- IOCの列挙
    - IOCはこちら: https://www.trendmicro.com/en_us/research/26/e/quasar-linux-qlnx-a-silent-foothold-in-the-software-supply-chain.html
- 推奨事項
    - Trend Microが提供するIoCを使い、QLNX感染の検出と防御を行う。
    - Trend Vision Oneのハンティングクエリでquasar-implantやPAM/LD_PRELOAD関連アーティファクトを探索する。
    - HTTP/TCPのBackdoor.Linux.QLNX.Aランタイム検知を活用する。
- その他
    - 攻撃者の帰属は不明で、狙いは開発者・DevOps認証情報を窃取し、サプライチェーン攻撃へつなげることと記載されている。
    - 一次ソース: https://www.trendmicro.com/en_us/research/26/e/quasar-linux-qlnx-a-silent-foothold-in-the-software-supply-chain.html
- ChatGPTの推奨事項
    - 開発者Linux端末でquasar-implant、/etc/ld.so.preload、PAM改変、ip-api[.]com通信、開発者シークレット窃取痕跡を今すぐ調査してください。
