#### Tycoon2FA、デバイスコードフィッシングでMicrosoft 365アカウントを乗っ取り
https://www.bleepingcomputer.com/news/security/tycoon2fa-hijacks-microsoft-365-accounts-via-device-code-phishing/

- 要約
    - Tycoon2FAフィッシングキットは、デバイスコードフィッシングに対応し、Microsoft 365アカウントの乗っ取りに悪用されている。
    - 攻撃は請求書を装ったメール内のTrustifiクリック追跡URLから始まり、複数のリダイレクトと難読化JavaScriptを経由する。
    - 被害者は偽のMicrosoft CAPTCHAページでOAuthデバイスコードを取得し、正規のMicrosoftデバイスログインページに入力するよう誘導される。
    - 被害者がMFAを完了すると、Microsoftは攻撃者管理デバイスにOAuthアクセス・リフレッシュトークンを発行する。
    - Tycoon2FAは研究者、スキャナ、VPN、サンドボックス、AIクローラなどを検知・回避する保護機能も備えている。
- IOCの列挙
    - IOCはこちら: https://github.com/eSentire/iocs/blob/main/Tycoon2FA/Tycoon2fa-iocs-03-23-2026.txt
- 推奨事項
    - 不要な場合はOAuthデバイスコードフローを無効化する。
    - OAuth同意権限を制限し、サードパーティアプリに管理者承認を要求する。
    - Continuous Access Evaluationを有効化し、準拠デバイスのアクセス制御とEntraログ監視を実施する。
- その他
    - 脆弱性に関する記事ではなく、Tycoon2FAによるMicrosoft 365アカウント乗っ取り手法に関する記事。
    - 攻撃者の国・動機などの属性情報は記載なし。
    - 一次ソース: https://www.esentire.com/blog/tracking-tycoon2fa-device-code-variant
- ChatGPTの推奨事項
    - Entra IDでOAuthデバイスコードフローの利用状況、deviceCode認証、Microsoft Authentication Broker、Node.jsユーザーエージェントを直ちに確認する。

#### Microsoft、重大なAzure脆弱性報告を却下しCVEは発行されず
https://www.bleepingcomputer.com/news/security/microsoft-rejects-critical-azure-vulnerability-report-no-cve-issued/

- 要約
    - 研究者は、Azure Backup for AKSで低権限のBackup Contributorからcluster-adminを得られる重大な権限昇格を報告した。
    - Microsoftはこの報告を脆弱性ではなく想定動作だとして却下し、CVEやCVSSスコアは発行されなかった。
    - この脆弱性は、バックアップ拡張機能に「Trusted Access」を付与する際の仕組みを悪用し、自動的に特権的な関係を構築させてしまう問題（CWE-441）に分類される。
    - CERT/CCは脆弱性を検証しVU#284781を割り当てたが、CNA階層ルールによりケースは閉じられた。
    - 研究者は、報告後に新たな権限チェックが追加され、元の攻撃経路が使えなくなったためサイレント修正だと主張している。
    - CVEや勧告がないため、防御側は影響期間や修正時期を追跡しにくい問題があると記事は指摘している。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 推奨事項なし
- その他
    - 脆弱性に関する記事であり、実攻撃の記載はなく、研究者の発見・報告とMicrosoftの否定、サイレント修正疑惑が中心。
    - 一次ソース: https://olearysec.com/research/azure-backup-aks-silent-patch/
- ChatGPTの推奨事項
    - Azure Backup for AKSのBackup Contributor権限付与履歴を確認し、対象AKSクラスターのバックアップ・復元操作と権限変更を監査する。

#### ロシアのハッカー、Kazuarバックドアをモジュール型P2Pボットネットへ進化
https://www.bleepingcomputer.com/news/security/russian-hackers-turn-kazuar-backdoor-into-modular-p2p-botnet/

- 要約
    - ロシア系ハッカー集団Secret Blizzardは、Kazuarバックドアを長期潜伏・隠密性・情報収集向けのP2Pボットネットへ発展させた。
    - Microsoftの分析では、新しいKazuarはKernel、Bridge、Workerの3モジュールで構成され、感染環境内で役割を分担する。
    - Kernelはリーダーを選出し、非リーダー端末をサイレント化することで、C2との外部通信を減らして検知面を縮小する。
    - Workerはキー入力記録、スクリーンショット、ファイル収集、偵察、メール/MAPIデータ収集、最近使ったファイル窃取などを行う。
    - Kazuarは150種類の設定項目を持ち、AMSI、ETW、WLDPのバイパスなど多様な回避機能も備えている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - 企業は静的シグネチャではなく、Kazuarのモジュール型で高度に設定可能な性質を踏まえた振る舞い検知に重点を置く。
- その他
    - 攻撃者はロシア情報機関FSBと関連付けられるSecret Blizzardで、政府・外交組織、防衛関連組織、欧州・アジア・ウクライナの重要システムを標的とする。
    - 一次ソース: https://www.microsoft.com/en-us/security/blog/2026/05/14/kazuar-anatomy-of-a-nation-state-botnet/
- ChatGPTの推奨事項
    - KazuarのIPC、単一リーダーによる外部通信、HTTP/WebSocket/EWS経由C2、AMSI/ETW/WLDP回避の挙動検知ルールを直ちに確認・追加する。

#### Grafana Labsのセキュリティ侵害、ハッカーがGitHubにアクセスしコードベースをダウンロード
https://cybersecuritynews.com/grafana-labs-security-breach/

- 要約
    - Grafana Labsは、権限付きトークンを入手した不正な第三者がGitHub環境へアクセスし、同社のコードベースをダウンロードしたと公表した。
    - 侵入は多数配置されたカナリアトークンの発火で検知され、原因はpull_request_targetの設定不備で動くGitHub Actionの「Pwn Request」脆弱性だった。
    - 攻撃者はリポジトリをフォークし、curlコマンドで悪意あるコードを注入して環境変数を暗号化ファイルへ出力し、権限付きトークンを窃取した。
    - 攻撃者はフォークを削除して痕跡を隠し、さらに4つの非公開リポジトリに同様の攻撃を行った後、コード公開を盾に身代金を要求した。
    - Grafana Labsは顧客データや個人情報へのアクセス、顧客システムや運用への影響を示す証拠はないと述べている。
- IOCの列挙
    - IOC情報なし
- 推奨事項
    - FBIの公開ガイダンスとして、身代金の支払いはデータ返還を保証せず、犯罪行為への参加を促す誘因になると記載。
- その他
    - GitHub Actionの「Pwn Request」脆弱性または設定不備に関する記事であり、既に侵入、トークン窃取、コードベースのダウンロード、恐喝が行われている。
    - 攻撃者の国などの属性は不明。動機は、盗んだコードを公開しないことと引き換えに支払いを要求する恐喝。
    - 一次ソース: https://twitter.com/grafana/status/2055827123236171827?ref_src=twsrc%5Etfw
- ChatGPTの推奨事項
    - GitHub Actionsのpull_request_target利用ワークフローを直ちに棚卸しし、シークレット露出の有無と不審なCI実行履歴を確認する。
